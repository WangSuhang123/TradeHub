一、网站需求分析与数据库设计

1. 需求分析

TradeHub是一个校园二手交易平台，旨在为在校大学生提供一个安全、便捷的闲置物品交易渠道。随着大学生消费观念的转变，校园内闲置物品的流转需求日益增长。传统的线下交易存在信息不对称、交易效率低下等问题。本平台通过互联网技术，将买家与卖家有效连接，实现商品发布、浏览、购买、订单管理等一站式服务。系统采用B/S架构，分为前台用户系统和后台管理系统两部分。前台支持用户注册登录、商品分类浏览、关键词搜索、购物车管理、订单提交与跟踪、收藏商品等功能；后台支持管理员对用户、商品、分类、轮播图等进行统一管理。通过该平台，可以实现校园资源的循环利用，促进绿色校园建设。

二、网站模块详细设计

TradeHub校园二手交易平台采用B/S（Browser/Server）架构，基于Java EE技术体系开发。系统整体采用MVC（Model-View-Controller）设计模式，以Servlet作为控制器层接收请求并调用业务逻辑，JSP作为视图层展示页面内容，DAO（Data Access Object）和Service层作为模型层负责数据持久化和业务逻辑处理。

系统分为前台用户系统和后台管理系统两大模块。前台面向所有校园用户，提供商品浏览、搜索、购买等核心交易功能；后台面向管理员，提供系统管理和数据维护功能。BaseServlet基类通过Java反射机制实现方法级请求路由，使得每个Servlet可以通过URL参数"method"指定具体调用哪个方法，简化了Servlet的数量和管理。

三、网站功能实现

1. 首页界面

首页是用户进入系统后看到的第一个页面，集成了平台的核心入口。页面顶部包含导航栏，提供Logo品牌标识、商品搜索框、购物车入口、用户登录/注册按钮以及用户菜单（个人中心、我的订单、我的收藏、发布商品、后台管理、退出登录）。中间区域展示轮播广告图，支持自动轮播与手动切换。下方显示商品分类快捷入口和"最新发布"商品列表（每行3个商品卡片），侧边栏展示"热门推荐"排行榜。商品卡片包含图片、名称、分类标签、成色、价格和折扣信息。整个页面采用Tailwind CSS框架实现响应式布局，适配不同屏幕尺寸。

首页界面核心代码：

// IndexServlet.java - 首页数据加载 public class IndexServlet extends BaseServlet { private ProductService pService = new ProductService(); private CarouselService carouselService = new CarouselService(); public String index(HttpServletRequest request, HttpServletResponse response) { request.setAttribute("latestList", pService.getLatestList(1, 12)); request.setAttribute("hotList", pService.getHotList(10)); request.setAttribute("carouselList", carouselService.getActive()); return "/index.jsp"; } }

2. 用户登录注册界面

登录页面提供学号和密码的登录表单。用户输入学号和密码后，系统通过UserService进行身份验证，验证成功后将会话信息存入Session并跳转至首页。若账号已被管理员封禁，则提示"您的账号已被封禁"；若凭证错误则提示"学号或密码错误"。注册页面需要填写用户名、学号、密码、确认密码、姓名、手机号、学校、邮箱、QQ等信息，系统会校验证两次密码是否一致，以及学号和用户名是否已被注册。注册成功后提示跳转至登录页。

用户登录核心代码：

// UserServlet.java - 登录方法 public String login(HttpServletRequest request, HttpServletResponse response) { if ("GET".equalsIgnoreCase(request.getMethod())) { return "/user_login.jsp"; } String studentId = request.getParameter("student_id"); String password = request.getParameter("password"); User user = uService.login(studentId, password); if (user == null) { request.setAttribute("failMsg", "学号或密码错误，请重新登录！"); return "/user_login.jsp"; } if (user.getStatus() == 0) { request.setAttribute("failMsg", "您的账号已被封禁，请联系管理员！"); return "/user_login.jsp"; } request.getSession().setAttribute("user", user); return "redirect:/index"; }

3. 商品列表与搜索界面

商品列表页面展示所有在售商品，支持按分类筛选，分页展示（每页10条），并提供"最新"、"价格从低到高"、"价格从高到低"三种排序方式。每个商品以卡片形式展示图片、名称、分类、成色、价格、浏览量和折扣标签。商品搜索功能允许用户输入关键词，并可按分类、价格区间进行筛选，搜索结果同样支持分页和排序。用户点击商品卡片即可进入商品详情页。

商品列表核心代码：

// ProductServlet.java - 商品列表 public String list(HttpServletRequest request, HttpServletResponse response) { int categoryId = 0; int pageNumber = 1; String sort = request.getParameter("sort"); if (request.getParameter("categoryId") != null) categoryId = Integer.parseInt(request.getParameter("categoryId")); if (request.getParameter("pageNumber") != null) pageNumber = Integer.parseInt(request.getParameter("pageNumber")); if (sort == null) sort = "newest"; Page p = pService.getProductPage(categoryId, pageNumber, sort); request.setAttribute("p", p); request.setAttribute("categoryId", categoryId); request.setAttribute("sort", sort); return "/product_list.jsp"; }

4. 商品详情与发布界面

商品详情页面展示商品的完整信息，包括商品大图、名称、价格（含原价和折扣）、成色描述、分类标签、浏览量、商品描述文本以及卖家信息。登录用户可以对商品进行"加入购物车"、"立即购买"、"收藏"等操作。商品发布页面允许已登录用户通过富文本表单发布商品，需要填写商品名称、售价、原价、分类、成色、描述并上传商品图片。上传的图片由服务器自动重命名为UUID格式存储。发布成功后跳转至商品列表。

商品发布核心代码（文件上传部分）：

// ProductServlet.java - 商品发布（文件上传） String fileName = part.getSubmittedFileName(); if (fileName != null && !fileName.isEmpty()) { String uploadPath = request.getServletContext().getRealPath("/picture/"); String uuidName = UUID.randomUUID().toString() + fileName.substring(fileName.lastIndexOf(".")); File uploadDir = new File(uploadPath); if (!uploadDir.exists()) { uploadDir.mkdirs(); } part.write(uploadPath + uuidName); product.setImage("picture/" + uuidName); }

5. 购物车与订单界面

购物车页面展示用户已添加的商品列表，包括商品图片、名称、单价、数量（可增减）、小计金额。支持单选/全选商品、批量删除、清空购物车等操作，底部显示已选商品总金额和"去结算"按钮。订单提交页面需要用户确认联系人、电话、交易地址等信息并填写卖家留言，提交后生成订单记录。订单列表页面分为"我是买家"和"我是卖家"两个Tab页，分别展示用户作为买家或卖家的订单，支持按订单状态筛选（待付款/已付款/已发货/已完成/已取消）。订单状态流转为：买家下单→待付款→买家付款→已付款→卖家发货→已发货→买家确认收货→已完成。

订单确认核心代码：

// OrderServlet.java - 订单确认 public String confirm(HttpServletRequest request, HttpServletResponse response) { User user = (User) request.getSession().getAttribute("user"); if (user == null) return "redirect:/user?method=login"; int productId = Integer.parseInt(request.getParameter("productId")); Product product = pService.getById(productId); if (product.getUserId() == user.getId()) { request.setAttribute("failMsg", "不能购买自己发布的商品"); return "/order_submit.jsp"; } Order order = new Order(); order.setTotal(product.getPrice()); order.setStatus(0); order.setBuyerId(user.getId()); order.setSellerId(product.getUserId()); // ... 设置订单明细 ... oService.addOrder(order); return "redirect:/order?method=myOrders"; }

6. 后台管理系统界面

后台管理系统仅供管理员角色访问（isadmin=1），通过AdminFilter过滤器进行权限校验。后台首页展示系统关键数据。用户管理页面展示所有用户列表，管理员可以对违规用户执行封禁/解封操作。商品管理页面展示全部商品列表，管理员可对违规商品执行下架操作。分类管理页面支持分类的增删改查。轮播图管理页面支持首页轮播广告的上传、编辑和排序。

后台管理核心代码（分类管理）：

// AdminServlet.java - 分类管理 public String categoryAdd(HttpServletRequest request, HttpServletResponse response) { String name = request.getParameter("name"); String description = request.getParameter("description"); Category category = new Category(); category.setName(name); category.setDescription(description); cService.add(category); return "redirect:/admin?method=categoryList"; } public String categoryDelete(HttpServletRequest request, HttpServletResponse response) { int id = Integer.parseInt(request.getParameter("id")); cService.delete(id); return "redirect:/admin?method=categoryList"; }

四、实验总结

通过本次Web应用开发实训，我成功设计并实现了一个功能完善的校园二手交易平台——TradeHub。在项目开发过程中，我深入实践了Java EE的Web开发技术体系，将课堂所学理论知识应用于实际项目开发中。

技术收获方面：我掌握了Servlet + JSP的MVC开发模式，深入理解了BaseServlet通过反射机制实现方法级路由的设计思想。通过使用C3P0连接池和Apache Commons DbUtils，我学会了如何高效地进行数据库操作和管理数据库连接。在数据库设计方面，我实践了MySQL的关系型数据库表结构设计、主外键约束、唯一索引等技术，设计了8张数据表来支撑整个业务系统。在前端方面，我使用JSTL标签库实现动态数据渲染，使用Tailwind CSS框架实现现代化响应式界面设计，使用原生JavaScript实现了轮播图、购物车交互等功能。此外，我还实践了文件上传（Servlet 3.0的Part API）、过滤器（编码过滤和管理员权限过滤）、监听器（ApplicationListener初始化数据）等Java Web高级特性。

遇到的问题及解决方案：（1）web.xml配置问题：项目部署后出现404错误，经排查发现IDEA的Web Facet指向了一个空的web.xml文件，而正确的web.xml在模块源码目录下。解决方案是重新配置Facet的Deployment Descriptor路径指向正确的web.xml。（2）BaseServlet方法分发：通过反射调用方法时，若method参数对应的实际方法返回类型为void，则不需要进行结果转发处理。修改BaseServlet的service方法，增加对void返回类型的判断。（3）商品图片上传：处理多部分表单数据时，需要正确区分普通表单字段和文件字段，通过判断Part.getSubmittedFileName()是否为空来区分。（4）订单状态流转：设计订单状态机时，需要在每个状态变更操作前验证当前状态是否符合变更条件，防止非法状态跳转。（5）购物车去重：同一用户对同一商品重复添加时，应更新数量而非插入新记录，通过在cart表中设置user_id+product_id的联合唯一索引实现。

项目展望：当前系统已实现基本的二手交易功能，未来可以进一步扩展以下功能：（1）引入在线即时通讯（WebSocket），支持买卖双方实时沟通；（2）增加商品评价与评分系统；（3）接入第三方支付接口（如微信支付、支付宝）；（4）增加消息通知机制，如订单状态变更通知；（5）引入Elasticsearch实现更强大的全文搜索功能。

综上所述，本次实训让我对Java Web开发有了全面深入的理解，不仅提升了编程实践能力，也培养了分析问题、解决问题的工程思维。项目虽已基本完成核心功能，但仍有许多优化空间，今后将继续学习完善。
